# De-EMR-AI-Agent

## 專案敘述

本專案致力於開發應用於**醫療場域的智慧代理人系統（AI Agent）**。 </br>
系統核心架構建立在對**健檢指標的深度解析**之上，回應臨床情境中「單一指標具有多重解釋可能性」的實務需求。

## 領域知識表（Domain Table）設計

鑑於同一健檢指標在不同臨床觀點與情境下具備多重解釋性，本系統建構了一套完整的**領域知識表**，並採用**三層式架構**進行設計：

<img width="365" height="208" alt="image" src="https://github.com/user-attachments/assets/b2188aec-f198-4f4f-8a48-7b8598a24fa4" />

#### 第一層：尺度層級
- 宏觀（Macro）
- 微觀（Micro）

#### 第二層：分析面向
- 本體（Ontology）
- 客體（Object）
- 環境（Environment）

#### 第三層：認知層次
- 觀察（Observation）
- 理解（Interpretation）
- 符號（Symbol）

透過上述三個維度的交織組合，系統共形成 **18 種屬性類別**，並以**自建資料庫**的方式，將各項健檢指標與其子屬性進行精確分類與管理

## 推論機制設計

在推論流程上，系統採取由下而上的多階段推論策略：

1. **微觀推論（Micro-level Reasoning）**  
   系統首先根據多維度的解釋狀態進行微觀推論，用以辨識健檢指標狀態改變的具體成因，並將結果彙整為一組「**原因集合**」。

2. **意識連結建構（Conscious Links）**  
   接著，系統於各成因之間建立具備宏觀邏輯推演能力的**意識連結（Conscious Links）**，以形成更高層次的因果理解。

3. **檢索增強生成（RAG）整合**  
   最後，系統導入**檢索增強生成（Retrieval-Augmented Generation, RAG**技術，  
   從專業醫療資料庫中擷取關鍵資訊以強化 LLM 的提示工程（Prompt Engineering）。

## 系統輸出成果

透過整合後的資訊脈絡與推論結果，系統能夠向使用者輸出：

- 精準的**診斷解釋**
- 具臨床價值的**指標意義說明**
- 可理解、可追溯的**推論脈絡**

以協助醫療人員與使用者更全面地理解健檢指標所隱含的臨床意義
